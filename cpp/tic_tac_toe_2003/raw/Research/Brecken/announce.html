 About this manual

This document was created for a technical writing class. I decided to put it on the web to see if anyone would find it useful. So far, a lot of people have!

 May 12, 1999

Minor update - I fixed some broken links on the source code pages, and updated the download page.

 April 25, 1999

Hello. Okay, so it's been more than a year and a half since the last change! This will probably be the last major update for the site. I considered taking it down, since VGA programming in kinda out-of-date, but I realize that VGA programming is a good way to learn graphics programming, and that some schools and universities teach it in their graphics courses. Here's what's changed since the last update:

  • Created DJGPP versions of all the code.
  • Updated the FAQ.
  • Fixed stack overflow problem in mouse.c.
  • New download options.
  • Updated links.
  • Edited some of the text.
  • New look, and the HTML now works better in older browsers.

Thanks to everyone who has ever written congratulations, or pointed out a problem, or suggested a solution. You've really helped a lot.

See ya,
Dave

 October 9, 1997

Hey everybody. I've been real busy with school and stuff. Nancy Sia pointed out a bug in the bitmap-reading code. Some paint programs store less than 256 palette entries in a 256-color bitmap, but the code always assumed there were 256 palette entries there. You can imagine what problems that can bring. The code is fixed now, though! That's the only update this time.

 August 1, 1997

Here's a small update for the site. Here's what I've done:

  • Expanded the Other Resources section.
  • Now you can download source code, executables, and bitmaps in convenient zip files. (This also eliminates the "Server Error" that occurs when you try to download ".exe" files.)
  • The .c files have been color coded! Woo hoo! Life is good.
  • For those of you who are still using Netscape 3.0 (hurry up and upgrade to Netscape 4.0!) the navigation bar will be a little easier to read.

That's about it. Comments are welcome, but let me say one thing. Please do not rearrange, modify, or add code to the code at the site and then ask me to debug it! This is happening waaay too much. Debug your own code. I will, however, be happy to help you if the code from this site doesn't compile on your compiler. Check out the Trouble Shooting section!

 March 31, 1997

Once again, it's been a while since I updated the site! This time the look and navigation has changed, plus this site now works better with Navigator 4.0's implementation of style sheets. Both Microsoft's and Netscape's implementations of Cascading Style Sheets are different, but the results are similar.

I took out the responses section and the hexadecimal reference page -- they were both pretty useless. I added a few links to the main page. Two new sections were added: Compiler Information, which talks about specific compilers (16- and 32-bit) and how to program the VGA with them, and I added to the Unchained Mode section, which covers the different modes you can get (like 320x240) by tweaking the VGA's registers.

Oh yeah. Today's my birthday! I'm a whopping twenty-two years old.

 February 19, 1997

Okay, it's been a while, but this site has finally been updated! Here's what's been added:

  • Responses, which is a collection of email messages sent to me about this site.
  • This entire manual can now be downloaded for off-line browsing.
  • New section added! Double Buffering, Page Flipping, and Unchained Mode

Here's what's been fixed and changed:

  • At the beginning of the Bitmaps and Palette Manipulation, there was a broken link to an image.
  • In the middle of Bitmaps and Palette Manipulation, a paragraph was added on how the BMP format pads its scanlines to four bytes, and how the code present does not reflect that fact.
  • At the end of the Bitmaps and Palette Manipulation, it was incorrectly stated that the VGA's refresh rate is 60hz. This is true in some modes, but in mode 13h, the refresh rate is 70hz.
  • In the appendix, the hexadecimal-to-binary table was actually a decimal-to-binary table.
  • Added to the introduction about a quirk in Turbo C compilers.
  • All of the black-and-white images were converted from 8-bit color to 4-bit color in order to save a little space and decrease download time.
  • Almost everything on this site is done with style sheets now, so if you don't have a Cascading Style Sheet (CSS-1) capable browser, things may look a little different. The content is still be readable by every browser. If you do have a CSS-1 capable browser, you'll see different colors, different fonts, and proper formatting.

Remember, if anyone finds any errors or broken links, or if you want to see anything in an upcoming section, email me!
